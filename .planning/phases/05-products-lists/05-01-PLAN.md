  <action>
Create custom archive/unarchive tools for products and bundles, following the exact pattern from src/tools/custom/contact-archive.ts.

Products and bundles both use `PATCH { is_archived: boolean }` for archive/unarchive (confirmed in product.yaml lines 127-152 for products, lines 261-283 for bundles). This is the same pattern as contacts.

Create a `registerProductArchiveTools` function that registers 4 tools:
1. `archive-product` - PATCH /products/{id} with { is_archived: true }
   Description: "Archive a product (hide from active lists). Use this for products with transaction history instead of deleting them."
2. `unarchive-product` - PATCH /products/{id} with { is_archived: false }
   Description: "Unarchive a product (restore to active lists)."
3. `archive-product-bundle` - PATCH /products/bundles/{id} with { is_archived: true }
   Description: "Archive a product bundle (hide from active lists). Use this for bundles with transaction history instead of deleting them."
4. `unarchive-product-bundle` - PATCH /products/bundles/{id} with { is_archived: false }
   Description: "Unarchive a product bundle (restore to active lists)."

Each tool:
- Takes a single `id` parameter (z.number())
- Calls client.patch with the appropriate path and body
- Returns JSON.stringify(result, null, 2) on success
- Uses transformHttpError / transformNetworkError for error handling
- Logs registration via log()

The function signature: `registerProductArchiveTools(server: McpServer, client: BukkuClient): number`
Returns 4 (the number of tools registered).

Import McpServer, z, BukkuClient, transformHttpError, transformNetworkError, log following the same import pattern as contact-archive.ts.
