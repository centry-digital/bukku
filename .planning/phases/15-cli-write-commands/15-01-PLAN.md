1. Create `packages/cli/src/input/json.ts` with a `readJsonInput` function:
   - Accepts `opts: Record<string, unknown>` parameter
   - If `opts.data` exists (string), parse it with `JSON.parse()` and return the result
   - If `!process.stdin.isTTY` (stdin is piped), read all stdin data, parse as JSON, return result
   - If neither: call `outputError({ error: 'JSON input required. Use --data flag or pipe JSON to stdin', code: 'VALIDATION_ERROR' }, ExitCode.VALIDATION)`
   - Wrap JSON.parse in try/catch; on SyntaxError call `outputError({ error: 'Invalid JSON input: ' + err.message, code: 'VALIDATION_ERROR' }, ExitCode.VALIDATION)`
   - For stdin reading, use: `const chunks: Buffer[] = []; for await (const chunk of process.stdin) { chunks.push(chunk as Buffer); } return JSON.parse(Buffer.concat(chunks).toString('utf-8'))`
   - Export signature: `export async function readJsonInput(opts: Record<string, unknown>): Promise<unknown>`

2. Extend `packages/cli/src/commands/factory.ts` — add three new functions and wire them into `registerEntityCommands`:

   **`addCreateCommand(resourceCmd, config)`:**
   - Command: `resourceCmd.command('create').description('Create a new ' + config.description)`
   - Option: `--data <json>` with description "JSON data (or pipe to stdin)"
   - Action uses `withAuth`: calls `readJsonInput(opts)` to get parsed body, then `client.post(config.apiBasePath, body)`, then `outputJson(data)`
   - Import `readJsonInput` from `../input/json.js`

   **`addUpdateCommand(resourceCmd, config)`:**
   - Command: `resourceCmd.command('update <id>').description('Update a ' + config.description)`
   - Option: `--data <json>`
   - Action: parse ID same pattern as addGetCommand (parseInt, validate positive, stash via `setOptionValue`), call `readJsonInput(opts)`, then `client.put(config.apiBasePath + '/' + id, body)`, then `outputJson(data)`

   **`addDeleteCommand(resourceCmd, config)`:**
   - Command: `resourceCmd.command('delete <id>').description('Delete a ' + config.description)`
   - No --data option needed
   - Action: parse ID same pattern as addGetCommand, call `client.delete(config.apiBasePath + '/' + id)`, then `outputJson({})`

   **Wire into `registerEntityCommands` loop:**
   - After the existing `if (config.operations.includes('get'))` block, add:
   - `if (config.operations.includes('create')) { addCreateCommand(resourceCmd, config); }`
   - `if (config.operations.includes('update')) { addUpdateCommand(resourceCmd, config); }`
   - `if (config.operations.includes('delete')) { addDeleteCommand(resourceCmd, config); }`
  </action>
