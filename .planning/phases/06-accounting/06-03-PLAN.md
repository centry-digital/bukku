  <action>
  Create `src/tools/custom/journal-entry-tools.ts` that registers 2 custom tools: create-journal-entry and update-journal-entry. These tools wrap the standard create/update pattern but add double-entry validation before the API call.

  Imports:
  - McpServer from "@modelcontextprotocol/sdk/server/mcp.js"
  - z from "zod"
  - BukkuClient type from "../../client/bukku-client.js"
  - transformHttpError, transformNetworkError from "../../errors/transform.js"
  - log from "../../utils/logger.js"
  - validateDoubleEntry from "../validation/double-entry.js"

  Export function `registerJournalEntryTools(server: McpServer, client: BukkuClient): number`

  Tool 1: "create-journal-entry"
  - Description (per user locked decision -- must mention list-accounts for tool chaining): "Create a new journal entry. Use list-accounts to find valid account IDs for line items. Journal entries must have balanced debits and credits (total debits = total credits)."
  - Input schema: { data: z.record(z.string(), z.unknown()).describe("Journal entry data including journal_items array with debit/credit amounts and account IDs") }
  - Handler logic:
    1. Extract journal_items from params.data: check if params.data.journal_items exists and is an Array (use Array.isArray)
    2. If journal_items array found, run validateDoubleEntry(journal_items as array of {debit?, credit?})
    3. If validation fails, return { isError: true, content: [{ type: "text", text: validation.error }] } -- do NOT call API
    4. If validation passes (or no journal_items found -- let API validate field presence), proceed with client.post("/journal_entries", params.data)
    5. Return { content: [{ type: "text", text: JSON.stringify(result, null, 2) }] } on success
    6. Standard error handling (Response -> transformHttpError, else transformNetworkError)
  - NOTE: If journal_items is not found in data, do NOT block the request. The API will return its own error about missing required fields. We only validate balance when line items are present.

  Tool 2: "update-journal-entry"
  - Description: "Update an existing journal entry. If updating line items, journal entries must have balanced debits and credits (total debits = total credits)."
  - Input schema: { id: z.number().describe("The journal entry ID"), data: z.record(z.string(), z.unknown()).describe("Updated journal entry data") }
  - Handler logic:
    1. Same validation pattern as create: extract journal_items from params.data, validate if present
    2. If validation fails, return isError with error message
    3. If valid or no line items in update data, proceed with client.put(`/journal_entries/${params.id}`, params.data)
    4. Standard error handling

  Log each tool registration. Return 2.

  IMPORTANT: The validation only runs when journal_items (the line items array) is present in the data. Partial updates that don't touch line items should pass through without validation. The key name for line items may be "journal_items" -- check accounting.yaml if uncertain, but use "journal_items" as the default based on research patterns.
