<objective>
Fix double-entry validation field name mismatch so unbalanced journal entries are caught client-side with conversational errors.

Purpose: UAT test 4 revealed that validateDoubleEntry() reads `line.debit` and `line.credit`, but Bukku API journal_items use `debit_amount` and `credit_amount`. The validation always sees 0/0, making every entry appear balanced. Unbalanced entries pass through to the API and get rejected with raw errors instead of the conversational message with totals and difference.

Output: Fixed validation function and tests that correctly read `debit_amount`/`credit_amount` from journal entry line items.
</objective>
