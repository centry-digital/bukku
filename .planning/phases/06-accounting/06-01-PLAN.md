<feature>
  <name>Double-Entry Validation</name>
  <files>src/tools/validation/double-entry.ts, src/tools/validation/double-entry.test.ts</files>
  <behavior>
    validateDoubleEntry(lines, minLines?) validates journal entry line items.

    Interface:
    - Input: Array of objects with optional `debit` and `credit` number fields, optional minLines (default 2)
    - Output: { valid: boolean, error?: string }

    Cases:
    - [{debit: 100}, {credit: 100}] -> { valid: true }
    - [{debit: 1500}, {credit: 1450}] -> { valid: false, error: "Journal entry is unbalanced. Total debits: 1500.00, Total credits: 1450.00. Difference: 50.00. Debits must equal credits in double-entry accounting." }
    - [{debit: 100}] -> { valid: false, error: "Journal entries require at least 2 line items..." }
    - [] -> { valid: false, error: "Journal entries require at least 2 line items..." }
    - [{debit: 33.33}, {debit: 33.33}, {debit: 33.34}, {credit: 100}] -> { valid: true } (tests multi-line summing)
    - [{debit: 1.20}, {debit: 1.30}, {credit: 2.50}] -> { valid: true } (floating-point safe with 0.01 epsilon)
    - [{debit: 0}, {credit: 0}] -> { valid: true } (zero amounts valid -- API will validate if nonsensical)
    - [{}, {}] -> { valid: true } (missing debit/credit treated as 0)
    - [{debit: 100, credit: 50}, {credit: 50}] -> { valid: true } (line can have both debit and credit)
  </behavior>
  <implementation>
    Create `src/tools/validation/double-entry.ts` exporting:

    ```typescript
    interface JournalEntryLine {
      debit?: number;
      credit?: number;
      [key: string]: unknown;  // Allow other fields to pass through
    }

    interface ValidationResult {
      valid: boolean;
      error?: string;
    }

    export function validateDoubleEntry(
      lines: JournalEntryLine[],
      minLines: number = 2
    ): ValidationResult
    ```

    Implementation details:
    - Check lines.length < minLines first, return error with current count and minimum
    - Sum all debit values (treating undefined/missing as 0)
    - Sum all credit values (treating undefined/missing as 0)
    - Compare: Math.abs(totalDebits - totalCredits) < 0.01 (epsilon tolerance for floating-point)
    - If unbalanced, return error with .toFixed(2) formatted totals and difference
    - Error message tone: conversational, includes enough detail for Claude to self-correct

    Use Node.js built-in test runner (per Phase 1 decision). Test file uses `node:test` and `node:assert`.
  </implementation>
</feature>
