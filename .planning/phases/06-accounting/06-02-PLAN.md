  <action>
  Create `src/tools/custom/account-tools.ts` with 3 tools: search-accounts, archive-account, unarchive-account.

  Imports:
  - McpServer from "@modelcontextprotocol/sdk/server/mcp.js"
  - z from "zod"
  - BukkuClient type from "../../client/bukku-client.js"
  - transformHttpError, transformNetworkError from "../../errors/transform.js"
  - log from "../../utils/logger.js"

  Export function `registerAccountCustomTools(server: McpServer, client: BukkuClient): number`

  Tool 1: "search-accounts"
  - Description: "Search and filter accounts from the chart of accounts. Supports filtering by category and archived status. For a quick cached account lookup (e.g., to find account IDs for journal entries), use list-accounts instead."
  - Input schema:
    - search: z.string().optional().describe("Search by account name, code, or description")
    - category: z.enum(["assets", "liabilities", "equity", "income", "expenses"]).optional().describe("Filter by account category")
    - is_archived: z.boolean().optional().describe("Filter by archived status (default: false, showing only active accounts)")
    - sort_by: z.enum(["code", "name", "balance"]).optional().describe("Sort field (default: code)")
    - sort_dir: z.enum(["asc", "desc"]).optional().describe("Sort direction (default: asc)")
    - page: z.number().optional().describe("Page number for pagination")
    - page_size: z.number().optional().describe("Number of items per page")
  - Handler: client.get("/accounts", params) -- pass all params as query parameters
  - Standard error handling (Response -> transformHttpError, else transformNetworkError)
  - Return JSON.stringify(result, null, 2) on success

  Tool 2: "archive-account" (following contact-archive.ts pattern)
  - Description: "Archive an account (hide from active lists). Use this instead of delete for accounts with transaction history. Archived accounts are hidden by default but can be reactivated."
  - Input: { id: z.number().describe("The account ID") }
  - Handler: client.patch(`/accounts/${params.id}`, { is_archived: true })
  - Standard error handling

  Tool 3: "unarchive-account"
  - Description: "Unarchive an account (restore to active lists)."
  - Input: { id: z.number().describe("The account ID") }
  - Handler: client.patch(`/accounts/${params.id}`, { is_archived: false })
  - Standard error handling

  Log each tool registration. Return 3.
  </action>
