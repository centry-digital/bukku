**packages/cli/src/config/rc.ts** — ~/.bukkurc read/write:
- Export `RC_PATH` as `path.join(os.homedir(), '.bukkurc')`.
- Export `readRc(): Record<string, string>` — reads INI-format file (`key = value` lines, skip blank/comment lines starting with `#`). Return empty object if file doesn't exist. Use `node:fs/promises` readFile.
- Export `writeRc(key: string, value: string): void` — reads existing rc (or empty), sets key, writes back as INI lines. Creates file with `{ mode: 0o600 }`. Use writeFile with mode.
- Export `checkPermissions(): { ok: boolean; mode: string }` — stat the file, check `mode & 0o777 === 0o600`. Return ok=true if file doesn't exist (nothing to warn about). On Windows (`process.platform === 'win32'`), always return ok=true.
- Keep parsing simple: split lines on first `=`, trim key and value. No INI sections needed.

**packages/cli/src/config/auth.ts** — 3-tier auth resolution:
- Export interface `ResolvedAuth { apiToken: string; companySubdomain: string; source: Record<string, 'flags' | 'env' | 'rc'> }` — source tracks where each field came from individually (not a single source for both).
- Export `resolveAuth(flags: { apiToken?: string; companySubdomain?: string }): ResolvedAuth` — precedence: flags > env vars (`BUKKU_API_TOKEN`, `BUKKU_COMPANY_SUBDOMAIN`) > rc file (readRc, keys `api_token`, `company_subdomain`). Throws an error with code `AUTH_MISSING` if apiToken or companySubdomain is not found in any tier. The error should be a custom class or plain object that the withAuth wrapper (Plan 13-02) can catch.
- Export `maskToken(token: string): string` — if token length >= 8, show first 4 + `...` + last 4 chars. Otherwise return `****`.

**packages/cli/src/index.ts** — Commander.js program:
- Do NOT add `#!/usr/bin/env node` shebang — esbuild adds it via banner config (per Phase 12 decision to avoid double-shebang bug).
- Import `Command` from `commander`, `createLogger` from `core`.
- Read version from a `version` const (hardcode `'0.0.1'` for now — esbuild will inline package.json version in Phase 16, or read at runtime via `createRequire`). Actually, use `import { readFileSync } from 'node:fs'` and `import { fileURLToPath } from 'node:url'` to read `../package.json` at runtime for the version. This way `--version` always matches package.json.
- Create `program = new Command()` with `.name('bukku')`, `.version(version)`, `.description('Bukku accounting CLI')`.
- Add global options: `.option('--api-token <token>', 'API token')` and `.option('--company-subdomain <subdomain>', 'Company subdomain')`.
- Register command groups as empty commands (placeholders for Phase 14): `sales`, `purchases`, `banking`, `contacts`, `products`, `accounting`, `files`, `settings`. Each with `.description(...)`. Use `program.command('sales').description('Sales invoices, quotes, orders, credit notes, payments, refunds')` etc.
- Import and register `configCommand` from `./commands/config.js`.
- Call `program.parseAsync(process.argv)` and catch unhandled errors — log to stderr and exit with code 1.

**packages/cli/package.json** — add test script:
- Add `"test": "node --test --experimental-strip-types 'src/**/*.test.ts'"` to scripts (matching core's pattern).

**packages/cli/src/commands/config.ts**:
- Export `configCommand` as a `new Command('config').description('Manage CLI configuration')`.
- **config set** subcommand: `configCommand.command('set').description('Set a config value').argument('<key>', 'Config key (api_token, company_subdomain)').argument('<value>', 'Config value')`. Action handler:
  - Validate key is one of `['api_token', 'company_subdomain']`. If not, write structured JSON error to stderr `{"error": "Invalid config key", "code": "VALIDATION_ERROR", "details": {"valid_keys": ["api_token", "company_subdomain"]}}` and exit with code 4.
  - Call `writeRc(key, value)` from `../config/rc.js`.
  - Write success JSON to stdout: `{"ok": true, "key": key, "message": "Config updated"}`.
  - Check permissions after write with `checkPermissions()`. If not ok, log warning to stderr (not JSON, just a human warning about permissions).
- **config show** subcommand: `configCommand.command('show').description('Show resolved configuration')`. Action handler:
  - Read rc with `readRc()`, check env vars, check global options from `this.parent.parent.opts()` (Commander passes parent options).
  - For each of `api_token` and `company_subdomain`, show the resolved value and its source (flags/env/rc/not set).
  - Mask the api_token value using `maskToken()`.
  - Write JSON to stdout: `{"config": {"api_token": {"value": "bk_xx...xxxx", "source": "rc"}, "company_subdomain": {"value": "mycompany", "source": "env"}}}`.
  - If neither field is set, still output the JSON structure with `null` values and `"source": "not set"`.
