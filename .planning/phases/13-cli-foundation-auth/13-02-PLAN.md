**packages/cli/src/output/json.ts**:
- Export `outputJson(data: unknown): void` — calls `process.stdout.write(JSON.stringify(data, null, 2) + '\n')`. Use `null, 2` for human-readable JSON (automation consumers can parse either way).

**packages/cli/src/output/error.ts**:
- Export const `ExitCode = { SUCCESS: 0, GENERAL: 1, AUTH: 2, API: 3, VALIDATION: 4 } as const`.
- Export interface `CliError { error: string; code: string; details?: unknown }`.
- Export `outputError(err: CliError, exitCode: number): never` — writes `JSON.stringify(err)` + newline to stderr via `process.stderr.write(...)`, then calls `process.exit(exitCode)`.
- Export `mapExitCode(errorCode: string): number` — maps error code strings to exit codes: `AUTH_MISSING` -> 2, `API_ERROR` / `API_HTTP_ERROR` -> 3, `VALIDATION_ERROR` -> 4, default -> 1.

**packages/cli/src/output/table.ts**:
- Export `outputTable(data: unknown, _columns?: string[]): void` — placeholder that writes a warning `[bukku-cli] Table format will be available in a future release. Showing JSON.` to stderr via `console.error()`, then calls `outputJson(data)` to fall back to JSON output. This is a stub — real table implementation comes in Phase 14.

**packages/cli/src/commands/wrapper.ts**:
- Import `BukkuClient` and `BukkuClientConfig` from `core`.
- Import `resolveAuth` from `../config/auth.js`.
- Import `outputError`, `ExitCode`, `mapExitCode` from `../output/error.js`.
- Import `createLogger` from `core`.
- Export interface `CommandContext { client: BukkuClient; opts: Record<string, unknown> }`.
- Export `withAuth(handler: (ctx: CommandContext) => Promise<void>)` — returns an async function that:
  1. Gets merged options via Commander's `this.optsWithGlobals()` (Commander v14 passes the Command as `this` in action handlers — use a regular function, not arrow function, to preserve `this` context). Extract `apiToken` and `companySubdomain` from merged opts.
  2. Calls `resolveAuth({ apiToken, companySubdomain })` — catches auth errors and calls `outputError({ error: 'Authentication required. Set credentials via --api-token flag, BUKKU_API_TOKEN env var, or bukku config set', code: 'AUTH_MISSING', details: null }, ExitCode.AUTH)`.
  3. Creates `new BukkuClient({ apiToken: auth.apiToken, companySubdomain: auth.companySubdomain })`.
  4. Calls `await handler({ client, opts: mergedOpts })`.
  5. Wraps the handler call in try/catch. If error is a `Response` object (API error — check `error instanceof Response`), read the body with `await error.json()`, then call `outputError({ error: body.message || 'API error', code: 'API_ERROR', details: body.errors || null }, ExitCode.API)`. If error is generic, call `outputError({ error: error.message || 'Unknown error', code: 'GENERAL_ERROR' }, ExitCode.GENERAL)`.
  </action>
