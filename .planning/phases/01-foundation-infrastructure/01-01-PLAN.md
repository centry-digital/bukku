<task type="auto">
  <name>Task 2: Implement environment validation and Bukku HTTP client</name>
  <files>src/config/env.ts, src/client/bukku-client.ts, src/utils/logger.ts</files>
  <action>
1. Create `src/utils/logger.ts`:
   - Export a `log` function that writes to stderr (NEVER stdout -- this is a stdio MCP server)
   - Use `console.error()` internally
   - Prefix messages with "[bukku-mcp]" for easy identification in logs

2. Create `src/config/env.ts`:
   - Define Zod schema for required env vars: BUKKU_API_TOKEN (string, min 1), BUKKU_COMPANY_SUBDOMAIN (string, min 1)
   - Export `validateEnv()` function that calls `envSchema.safeParse(process.env)`
   - On failure: print a clear checklist to stderr (per user's locked decision -- "immediate crash with checklist"):
     ```
     Configuration Error

     Missing required environment variables:
       - BUKKU_API_TOKEN: (error message)
       - BUKKU_COMPANY_SUBDOMAIN: (error message)

     Setup checklist:
       1. Go to Bukku web app -> Control Panel -> Integrations
       2. Turn on API Access and copy the Access Token
       3. Set BUKKU_API_TOKEN=<your-token>
       4. Set BUKKU_COMPANY_SUBDOMAIN=<your-subdomain> (e.g., 'mycompany' from mycompany.bukku.my)
       5. Restart Claude Desktop
     ```
   - On failure: call `process.exit(1)`
   - Export the `Env` type via `z.infer`

3. Create `src/client/bukku-client.ts`:
   - Export a `BukkuClient` class that takes `Env` in constructor
   - Base URL: `https://api.bukku.my` (production, per intro.yaml)
   - Every request includes headers:
     - `Authorization: Bearer ${token}`
     - `Company-Subdomain: ${subdomain}`
     - `Accept: application/json`
     - `Content-Type: application/json` (for POST/PUT/PATCH)
   - Methods:
     - `get(path: string, params?: Record<string, string | number | undefined>)` -- appends query params, returns parsed JSON
     - `post(path: string, body: unknown)` -- sends JSON body
     - `put(path: string, body: unknown)` -- sends JSON body
     - `patch(path: string, body: unknown)` -- sends JSON body (for status updates)
     - `delete(path: string)` -- no body
   - Each method uses native `fetch` (Node 18+)
   - On non-ok response: throw the Response object (error transformation happens in the error layer, not here)
   - `validateToken()` method: makes GET request to `/contacts` with `page_size=1` (lightweight call to verify token works). If 401, log clear auth error to stderr and call `process.exit(1)`. Per locked decision: "Validate token on startup."
   - IMPORTANT: Never log the actual token value. Log "Bearer ***" if debugging headers.
  </action>
