<feature>
  <name>HTTP to MCP Error Transformation</name>
  <files>src/errors/transform.ts, src/errors/transform.test.ts</files>
  <behavior>
The `transformHttpError` function takes an HTTP status code, an optional response body (parsed JSON), and an operation description string. It returns an MCP error response object.

Test cases (input -> expected output patterns):

1. (401, any body, "create invoice") -> text contains "Bukku token" AND "BUKKU_API_TOKEN" AND suggests checking/restarting
2. (403, any body, "list contacts") -> text contains "permission" or "access" AND suggests checking permissions
3. (404, any body, "get sales invoice") -> text contains "couldn't find" AND suggests listing items
4. (400, { message: "Validation failed", errors: { "contact_id": ["required"], "date": ["invalid format"] } }, "create invoice") -> text contains BOTH "contact_id" and "date" errors (all at once, per locked decision)
5. (400, { message: "Something wrong" }, "update order") -> text contains the message, suggests next step
6. (422, { message: "Unprocessable", errors: { "items": ["at least one item required"] } }, "create invoice") -> handles like 400 validation
7. (500, any body, "list invoices") -> text contains "unexpected" AND suggests trying again
8. (503, any body, "create payment") -> text contains "unavailable" or "temporarily" AND suggests trying later
9. Network error (no status code), "list contacts") -> text contains "connect" or "network" AND suggests checking connection

Every returned object must have shape: { isError: true, content: [{ type: "text", text: string }] }
Every text must contain a suggested next action (per locked decision).
  </behavior>
  <implementation>
Export function `transformHttpError(status: number | null, body: unknown, operation: string)` that returns `{ isError: true; content: Array<{ type: "text"; text: string }> }`.

Pattern matching by status code:
- 401: Auth failure with prominent BUKKU_API_TOKEN callout
- 403: Permission error
- 404: Not found with suggestion to list
- 400/422: Parse `errors` field from body, show all validation errors together. If no errors field, show message.
- 500-599: Server error with retry suggestion
- null (network error): Connection failure

Also export a helper `transformNetworkError(error: unknown, operation: string)` for fetch failures (TypeError: Failed to fetch, etc).

Use Node.js built-in test runner (`node --test`) for tests. Add test script to package.json: "test": "node --test --experimental-strip-types 'src/**/*.test.ts'"

IMPORTANT: Install no test framework. Use Node.js built-in `node:test` and `node:assert`.
  </implementation>
</feature>
