Discretion decision: Hand-craft types from OpenAPI specs rather than auto-generate. Rationale: The OpenAPI specs use `$ref` to shared files that are not present in the repo (e.g., `./common_components/servers.yaml`, `./parameters/query/page.yaml`). Auto-generation would fail or produce incomplete output. Hand-crafting the ~10 shared types is straightforward and gives full control. The OpenAPI specs remain checked in as reference only.

1. Create `src/types/bukku.ts` with types derived from the OpenAPI specs:

   **Pagination (from resPaging schema in sales.yaml):**
   ```typescript
   export interface BukkuPaging {
     current_page: number;
     per_page: number;
     total: number;
   }
   ```

   **Paginated list response wrapper:**
   ```typescript
   export interface BukkuPaginatedResponse<T> {
     paging: BukkuPaging;
     [key: string]: T[] | BukkuPaging;  // e.g., "transactions", "contacts"
   }
   ```

   **Single item response wrapper:**
   ```typescript
   export interface BukkuSingleResponse<T> {
     [key: string]: T;  // e.g., "transaction", "contact"
   }
   ```

   **Error response (from general-error.yaml pattern):**
   ```typescript
   export interface BukkuErrorResponse {
     message: string;
     errors?: Record<string, string[]>;
   }
   ```

   **List query parameters (shared across all list endpoints):**
   ```typescript
   export interface BukkuListParams {
     page?: number;
     page_size?: number;
     sort_by?: string;
     sort_dir?: 'asc' | 'desc';
     search?: string;
     date_from?: string;  // YYYY-MM-DD
     date_to?: string;    // YYYY-MM-DD
     status?: string;
   }
   ```

   **CRUD entity configuration (for the factory pattern):**
   ```typescript
   export interface CrudEntityConfig {
     entity: string;           // e.g., "sales-invoice"
     apiBasePath: string;      // e.g., "/sales/invoices"
     singularKey: string;      // e.g., "transaction" (response wrapper key)
     pluralKey: string;        // e.g., "transactions" (list response wrapper key)
     description: string;      // e.g., "sales invoice"
     operations: CrudOperation[];
     hasStatusUpdate?: boolean;
     listFilters?: string[];   // extra filter params beyond common ones
   }

   export type CrudOperation = 'list' | 'get' | 'create' | 'update' | 'delete';
   ```
